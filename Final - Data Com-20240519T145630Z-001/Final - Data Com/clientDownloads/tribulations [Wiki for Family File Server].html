Some of my Trials and Tribulations

Morse Code

For the morse code, I went through many different libraries to try and get it working. Either I got exceptions or the code would run, but have no audio output. I tried solving this with saving the audio to a file, but that also did not work.

I first tried using pygame, but no luck. I then tried libSDL2-devel, as I was having issues with my Pygame installation compatibility. Due to this, I tried using a previous version, as was using python 3.11 (pip install pygame==2.1.0). That still didn't fix the issue, so I installed “sudo zypper install freetype2-devel”, before trying again This still didnt fix it…, so I tried again. “sudo zypper install libjpeg-turbo-devel libpng-devel SDL2_ttf-devel SDL2_image-devel SDL2_mixer-devel portmidi-devel”

After hours of troubleshooting, I gave up on this approach, and tried, numpty, but it also gave me no luck. Here, I was able to run the program, and create an audio file (with size), but there was no actual sound.

So I then tried “pip install sounddevice”, but needed “sudo zypper install portaudio-devel” for this libaray.

Failing this, I tried “import os”, but that too didn't work.

Finally, after much pain and suffering, I made use of “import subproecess” which created a sound.

Music Player

I originally tried going for a video playing system, but after many hours of the OS not having the capabilities to play video, and many library installs finally fixing it, I realised playing videos or such during studying would be too distracting. So I scrapped that decided to go with audio only.

Google Drive

I needed a cloud, 3rd party backup service (as offsite backups are the most reliable) for my solution. I also needed something that was free. I found google cloud API gave a 3 month free trial, and could fit into my existing program with little to no issues. All it required was setting up the account, with linking my gmail email to it.

Server and Client

I spent quite a while trying to find a solution to connect the client and server, while also being able to transfer files. I ended up using the solution from here to build this connection https://thepythoncode.com/code/send-receive-files-using-sockets-python .

I had been running the client and server on the same VM, so my next step was putting each on a separate VMs. This resulted in them not being able to connect to each other of course. After reading some online guides and trying to ping their IPs', I changed the Oracle VM configuration settings to Bridged Networking. As before each of the VMs not only couldn't see each other, but they were each given the same IP and treated as the same device (kinda).

It still wasn't connecting, so In diagnosing the issue, I turned off the firewalls. That worked, so I knew that was the issue. I turned them back on, and opened a port on 5001 to allow incoming connections. Issue solved.

I decided to implement the morse code and media playing capabilities as two separate files, so I could play around getting them working, without modifying or breaking the existing code files I had made.

Once all this was working, I went about fixing up my GUI a bit.
