Problem Statement

My identified problem has one main issue, and 3 sub-problems.

Main Problem

Whether you work on bleeding edge technology, normal office work, or are just a student, losing important files can be devastating. Whether it is losing an assignment due the next day, an important schematic that has taken months to create, or even family photos from years past, such a loss can result loss of finances, sanity, or time. Such data loss is common, and can easily happen. From a mistaken deletion, data corruption, to the device storing the data dying.

Sub-Problem 1

In the modern age, we have 10s-100s of sites and passwords we must remember. This often leads to using one password for every site, or adding a 1 to the end of the password. This is extremely unsecure, and can lead to dire financial consequences. For family units, who share passwords for sites such as Netflix, having to reset the password due to it being forgotten can be very annoying as it will log everyone else out.

Sub-Problem 2

When working or studying, it is easy to get distracted by loud neighbors, family talking in the next room, or even passing cars. This decreases productivity, and results in more time spent on something than was originally needed.

Sub-Problem 3

In a family, it is often crucial that parents are able to communicate with each other privately, without their children knowing what is being said. This could be due to planning their party, or content not appropriate for a younger audience. Sometimes, parents need to have such discussions even when the child is there. In the early childhood days, simply spelling words out A-P-P-L-E works, however as the child develops, such an avenue becomes moot.

Problem Solution

My solution is meant to be placed in a family communal work area. It consists of a server (can run headless if needed), and clients (the family members).

Main Solution

In order to combat this loss of data, I have implemented a local file server. This server can handle up to 5 client connections. Each family member can send and retrieve files from this server. The clients can do this by requesting a file list on the server. This connection makes use of socket/tcp connections. As only those on the network can access it, this level of security should suffice. This file server acts as an on-site backup. For the rules of backups, it is also a good idea to have an offshore backup. In this case, I have implemented the ability for the server to create a backup of everything on the server, and send it to Google Clouds Storage (Stored in a bucket). This will ensure that no-one in the family will ever lose any data.

Sub-Solution 1

The file system server will allow family members to store and retrieve passwords. These sites and passwords are for communal entertainment services such as DSTV, NETFLIX or SPOTIFY. As there will be no communication over the internet, and such sites are only for streaming, having the passwords sent and retrieved on the local network is not a big security concern. This allows the family to have a way to store and view shared passwords, while still allowing them to keep their personal passwords else where.

Sub-Solution 2

The server contains playlist folders with instrumental music. The client can retrieve these folders, select one, and the server will start playing it. The server should have some type of speaker in order to play the audio. This will allow the studying student or working salary man to drown out any distracting background noise, while not distracting them in turn.

Sub-Solution 3

The client is able to enter text on their program, which is sent to the server. This text is converted to Morse Code, which is subsequently turned into sounds (played on server speakers). In this manner, parents will be able to communicate mature topic with each other without their child listening in.
